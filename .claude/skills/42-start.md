# /42-start

Usage: `/42-start [プロジェクト名]`

新しい42プロジェクトを開始するときに実行する。

## Claudeへの指示

以下の手順で実行せよ：

1. `~/Downloads/42subject/` から対応するPDFを探して読む
   - プロジェクト名に対応するファイルを探す（例：amazing.pdf, codexion.pdf, rag.pdf等）

2. 以下を出力する：

### このプロジェクトが教えようとしていること（3点）
- 技術的な学習目標を箇条書きで

### PMとして理解すべきこと
- エンジニアと話すときに使える概念
- このプロジェクトがVisitas/monolyst/GovGovのどの場面に対応するか

### バイブコーディング実装方針
- 実装の骨格（何をどの順番で作るか）
- 「なぜこう設計するか」の説明ポイント
- 詰まりやすいポイントの予告

3. `tasks/todo.md` に以下を追記する：
```
## [プロジェクト名]
- [ ] 概念理解（/42-start 完了）
- [ ] 実装
- [ ] /42-submit 完了
- [ ] lessons.md 記録
```
