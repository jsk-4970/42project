# /42-explain

Usage: `/42-explain [概念名]`

技術概念をPMの視点で理解するために使う。

## Claudeへの指示

以下の構成で説明せよ：

### 1. 一言で言うと
- 技術者でない人間にも伝わる1文

### 2. 何が起きているか（図解・例え）
- コードや数式ではなく、日常の例え話で説明する
- できればAsciiで図を描く

### 3. なぜこれを学ぶか（42の意図）
- このプロジェクトで42が教えようとしていること

### 4. PMとして「エンジニアに聞けるようになること」
- 具体的な質問例を3つ出す
- 例：「このAPIのレイテンシが高い原因はどこですか？」

### 5. Visitasでの実務への応用
- 電子カルテ・訪問診療システムの文脈でこの概念がどう関係するか
- エンジニア採用面接でこの概念を活かす場面

### 6. 最低限覚えること（3点）
- 実装を始める前に頭に入れておくべき概念の核心
