# /42-submit

Usage: `/42-submit [プロジェクト名]`

提出前の最終チェックを実行する。

## Claudeへの指示

以下の順番でチェックせよ：

### 1. コード品質チェック（言語別）

**Cプロジェクトの場合：**
```bash
norminette [対象ファイル]
```
- エラーがあれば指摘して修正案を出す

**Pythonプロジェクトの場合：**
```bash
flake8 [対象ファイル]
```

### 2. コンパイル・実行確認

```bash
make re  # Makefileがある場合
# または
cc -Wall -Wextra -Werror [ファイル名]
```

### 3. 動作確認

- subjectに記載されているテストケースを実行
- エッジケース（空入力、最大値、異常値）を確認

### 4. 提出前チェックリスト出力

- [ ] norminette/flake8 クリア
- [ ] コンパイルエラーなし
- [ ] 基本動作確認
- [ ] エッジケース確認
- [ ] 不要なファイル（a.out, .DS_Store等）がリポジトリに含まれていないか

### 5. 学びの記録

以下を出力し、`tasks/lessons.md` に追記する：

```markdown
## [プロジェクト名]（[日付]）
- 技術的学び：[1文]
- PMとしての学び：[1文 - エンジニアに何を聞けるようになったか]
- Visitasへの応用：[任意]
```

### 6. git commit メッセージ提案

```
feat: complete [プロジェクト名]
```
